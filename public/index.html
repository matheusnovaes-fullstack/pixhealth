Como Interpretar os Dados

Status da Instituição

OK
Sistema operacional. Latência inferior a 2000ms e todos os endpoints respondendo normalmente.
LENTO
Degradação de performance. Latência entre 2000-5000ms ou alguns endpoints inacessíveis.
CRÍTICO
Problema grave. Latência superior a 5000ms, erro HTTP 5xx ou maioria dos endpoints fora.
ERRO
Sistema offline. Timeout em todas as requisições ou todos os endpoints inacessíveis.

Métricas de Performance

Latência
Tempo de resposta atual em milissegundos. Valores inferiores a 1000ms são ideais. Acima de 3000ms indica possível problema.
Baseline
Média histórica das últimas 30 medições. Serve como referência para detectar anomalias. Atualizado continuamente.
Proporção
Relação entre latência atual e baseline. Valor 1.0x = normal. Acima de 3x indica degradação significativa.
URLs Online
Quantidade de endpoints respondendo corretamente. Formato: online/total. Menos que 100% indica problema parcial.

Score de Severidade (QUANTO MAIOR, PIOR)

Como Funciona
Sistema de pontuação (0-100%) que mede a GRAVIDADE do problema. 0% = tudo normal, 100% = problema gravíssimo. Baseado em: código HTTP, latência absoluta, proporção vs baseline, persistência e quantidade de URLs afetadas.
80-100% - CRÍTICO
Problema grave confirmado. Requer ação imediata.
60-79% - ALTO
Problema confirmado. Investigar em 5 minutos.
40-59% - MODERADO
Degradação detectada. Monitorar evolução.
20-39% - BAIXO
Anomalia leve. Nenhuma ação necessária.
0-19% - NENHUM
Sistema operacional e saudável.

Classificação de Prioridade

P1 - CRÍTICO
Status crítico ou erro com severidade ≥70%. Ação: Alertar equipe imediatamente.
P2 - URGENTE
Problema confirmado com severidade ≥50%. Ação: Investigar em até 5 minutos.
P3 - ATENÇÃO
Lentidão com severidade ≥30%. Ação: Monitorar próximas verificações.
P4 - INFO
Sistema operacional ou baixa severidade. Ação: Nenhuma.

Indicadores Especiais

Protegido (HTTP 403)
Site possui proteção anti-bot (Cloudflare/WAF). Não indica instabilidade. Sistema está online, mas bloqueia requisições automatizadas.
Problema Isolado
Quando apenas uma instituição apresenta problema enquanto 75%+ das demais estão normais. Aumenta score de severidade.
Persistente
Problema detectado em 3+ verificações consecutivas. Indica que não é intermitência momentânea.
